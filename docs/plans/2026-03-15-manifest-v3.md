# Chrome Manifest V3 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Convert the extension to Chrome Manifest V3 with minimal behavioral change.

**Architecture:** The manifest will point to a background service worker entry script. That worker will import the existing background and shared modules, while the runtime code switches from page-action and MV2 script injection APIs to MV3-compatible action and scripting APIs.

**Tech Stack:** Chrome Extensions Manifest V3, plain JavaScript, service worker background, context menus, scripting API

### Task 2: Convert manifest and background bootstrapping

**Files:**
- Modify: `manifest.json`
- Modify: `js/background/main.js`

**Step 1: Update the manifest for MV3**

Change `manifest_version` to `3`, replace `page_action` with `action`, add `background.service_worker`, add `scripting` permission, and move `<all_urls>` into `host_permissions`.

**Step 2: Verify MV2-only fields are removed**

Remove the background `scripts` array, `persistent`, and the MV2 `content_security_policy` entry.

**Step 3: Add service worker imports**

Use `importScripts(...)` in `js/background/main.js` so the existing shared and background files still load in the right order.

### Task 3: Migrate runtime APIs to MV3 equivalents

**Files:**
- Modify: `js/background/chrome_page_action.js`
- Modify: `js/shared/highlighter.js`
- Modify: `js/shared/chrome_tabs.js`
- Modify: `js/background/chrome_context_menus_handler.js`

**Step 1: Replace page-action calls**

Switch `chrome.pageAction.*` usage to `chrome.action.*`, using enable/disable in place of show/hide and updating the context menu to `action`.

**Step 2: Replace script injection**

Switch the tab helper from `chrome.tabs.executeScript()` to `chrome.scripting.executeScript()`, preserving sequential file injection.

**Step 3: Remove service-worker-incompatible alert usage**

Replace background-side `window.alert(...)` calls with tab-side execution so the context menu validation messages still appear to the user.
